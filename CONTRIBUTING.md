## I Want To Contribute

1. Pick an issue from the [issues page](https://github.com/opeolluwa/send-file/issues)
2. Comment to get assigned
3. Fork the repo
4. Clone locally
5. Make changes (follow commit guidelines)
6. Open a PR to the `master` branch

> You must own the code you contribute and agree it can be licensed under the project terms.

## Reporting Bugs

### Before submitting

* Use latest version
* Confirm it’s not user error
* Check existing issues
* Gather:

  * Stack trace
  * Environment details
  * Reproduction steps

### Submit

* Open an [issue](https://github.com/opeolluwa/send-file/issues/new)
* Describe expected vs actual behavior
* Include reproducible steps

> ⚠️ Security issues: email **[adefemiadeoye@yahoo.com](mailto:adefemiadeoye@yahoo.com)** (do not open publicly)

---

## Suggesting Enhancements

* Ensure it doesn’t already exist
* Align with project goals
* Open an issue with:

  * Clear title
  * Detailed description
  * Expected behavior
  * Use case

## Style Guides

### Commit Messages

```
<type>[scope]: <description>
```

**Types:**

* `feat` – new feature
* `fix` – bug fix
* `refactor` – code changes
* `docs`, `style`, `test`, `chore`, etc.

**Example:**

```
feat(ui): add shadow to nav item
```
